Desarollo de Logíca en GRAFCET y LADDER

Se desarrolla la lógica de operación que debe ejecutar la cortadora automática para realizar el proceso de corte. Primero, se elabora un GRAFCET de descripción funcional, cuyo objetivo principal es explicar en palabras cada etapa del proceso. Posteriormente, a partir de la clasificación de acciones y receptividades, se construye el GRAFCET tecnológico, que sirve como base para definir las partes secuencial (enfocada en las transiciones) y combinacional (centrada en las acciones).

Con esta estructura definida, se implementa la programación en LADDER dentro de Studio 5000, siguiendo fielmente el GRAFCET desarrollado. Finalmente, se realiza la conexión con Logix Emulate, como se muestra en el video adjunto.

Integración SCADA con NX mediante Ignition en la Nube

Se desarrolló la conexión del sistema SCADA con NX utilizando un servidor alojado en Azure (Chile) con un contenedor de Ignition versión 8.3.1, accesible mediante IP en el puerto 8088.

Arquitectura del Sistema:

Se instalaron tres módulos clave: MQTT Distributor, MQTT Engine y MQTT Transmission (todos en estado connected), que permiten la conexión con un Ignition local que se comunica con NX mediante protocolo OPC UA.

Interfaz HMI Desarrollada:

Utilizando Ignition Designer se creó una HMI para la cortadora de ejes con dos ventanas:

  • Ventana Principal: Visualiza bandas transportadoras (entrada/salida), cortadora e indicadores con código de colores (rojo: apagado, verde: encendido) para sensores, mordazas, eje Z y disco de corte. Incluye botones de marcha, stop y tres opciones de longitud de corte.
  • Ventana de Históricos: Registra en tiempo real la posición de mordazas, velocidad del disco de corte y posición del eje Z.

Operación del Sistema:

La demostración muestra el proceso completo: selección de longitud de corte, avance de la pieza por banda de entrada hasta activación de sensores, sujeción con mordazas, activación del disco y descenso del eje Z para corte, liberación de mordazas, transporte por banda de salida y recolección final en sumidero.

Beneficio: Integración en la nube que permite monitoreo y control remoto del proceso de corte con sincronización en tiempo real entre SCADA y simulación NX.

Propuesta de Valor Agregado - Sistema MES para RAMFE

La propuesta de automatización para RAMFE incluye la reorganización de estaciones para mejorar el flujo de materiales, implementación de una celda robotizada con cortadora automática para eliminar cuellos de botella, y como valor agregado principal, la integración de un sistema MES (Manufacturing Execution System) para toda la compañía.

Componentes del Sistema MES

Se diseñaron dos plantillas de registro:

  1. Plantilla para Estaciones: Registra estación, fecha, horas de inicio/finalización, categoría de producto (sinfín corona, helicoidal o cónico), ID del producto, número de intentos de reproceso y estado de calidad (aprobado/reprobado).
  2. Plantilla para Fallas: Documenta estación, ID del equipo, fecha y hora de falla, y fecha y hora de reparación.

Resultados del Sistema:

Los datos simulados en Python y visualizados en Power BI muestran un OEE del 82% con:

  • Eficiencia: 94%
  • Calidad: 96%
  • Disponibilidad: 90%

El dashboard permite analizar:

  • Reportes de fallas (graves/leves) por estación
  • Tiempos de ciclo promedio por producto y estación
  • El tratamiento térmico es la estación con mayor tiempo en los tres productos
  • El sinfín corona presenta la mejor calidad en procesos
  • Tendencias de OEE por producto, estación e intervalos de tiempo

Beneficio: Mayor control operativo, identificación de áreas de mejora, optimización de eficiencia y mejor aprovechamiento del tiempo de producción.
